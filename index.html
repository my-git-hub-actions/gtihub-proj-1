Basic Git Commands

git init

Initializes a new Git repository

git add <file>

Stages changes for commit

git commit -m "message"

Commits staged changes with a descriptive message

Common Git Operations

git status

Shows the status of your working directory

git log

Displays commit history

git branch

Lists all local branches

Branch Operations

git branch

Lists all local branches

git checkout -b <branch-name>

Creates and switches to a new branch

git merge <branch-name>

Merges specified branch into current branch

Remote Operations

git remote add origin <repo-url>

Connects local repository to remote repository

git push -u origin main

Pushes commits to remote repository

git pull origin main

Fetches and merges changes from remote repository

Soft Reset

git reset --soft HEAD~1
  • ✅ Preserves modified files in staging area
  • ✅ Allows recommitting with new message
  • ✅ Safe operation - no data loss
# Example: Combining last commit with new changes git reset --soft HEAD~1 git add . git commit -m "Better commit message"

Hard Reset

git reset --hard HEAD~1
  • ❌ Discards all changes permanently
  • ❌ Resets working directory completely
  • ⚠️ Cannot recover discarded changes
# Example: Complete reset to previous commit git reset --hard HEAD~1 # Use with caution!

Git Rebase vs Reset

Git Rebase

Rewrites commit history by moving commits to a new base

# Moving feature branch to latest main git checkout feature git rebase main # Interactive rebase for last 3 commits git rebase -i HEAD~3
  • ✅ Creates cleaner, linear history
  • ✅ Integrates changes from other branches
  • ⚠️ Don't use on public/shared branches
  • Consider we have feature branch with multiple commits, when we want to merge them with main branch, then we need to checkout to feature branch and run git rebase main to move feature branch commits onto latest main. Here while moving using rebase we interactively merge the commits to single commit for cleaner commit history
  • Here while rebasing we will get multiple options like pick - keep commit as is , squash - combine with previous commit, reword - change commit message, drop - remove commit

Git Reset

Moves branch pointer to different commit

# Soft reset - preserve changes git reset --soft HEAD~1 # Hard reset - discard changes git reset --hard HEAD~1
  • ✅ Quick way to undo local commits
  • ✅ Can keep or discard changes
  • ⚠️ Hard reset permanently loses changes

Git Stash Operations

Basic Stash Commands

Save Changes to Stash

# Quick stash git stash # Stash with message git stash save "work in progress" # Include untracked files git stash -u # Include all files git stash -a

View & Apply Stashes

# List all stashes git stash list # Show stash contents git stash show # Apply and remove latest stash git stash pop # Apply but keep stash git stash apply

Common Use Cases

Switch branches with uncommitted work:

git stash save "WIP: feature" git checkout other-branch # do work git checkout original-branch git stash pop

Important Notes:

  • ✅ Use descriptive messages with stash save
  • ✅ Check contents before applying with stash show
  • ⚠️ Stashes are stored locally only
  • ⚠️ Stash clear removes all stashes permanently

Working with Remote Repositories

Basic Remote Commands

View Remotes

# List remotes git remote # Show remote details git remote -v # Show detailed info git remote show origin

Manage Remotes

# Add new remote git remote add origin https://github.com/username/repo.git # Change remote URL git remote set-url origin git@github.com:username/repo.git # Rename remote git remote rename origin upstream # Remove remote git remote remove upstream

Common Operations

# Download from remote git fetch origin # Fetch and merge changes git pull origin main # Push local changes git push -u origin main

Important Notes:

  • ✅ HTTPS or SSH URLs can be used
  • ✅ Origin is the conventional name for primary remote
  • ⚠️ Always verify remote URL before pushing
  • ⚠️ Set up authentication before pushing

Git Collaboration

Working with Collaborators

Access Management

# View repository collaborators (GitHub CLI) gh repo view [repository] --collaborators # Add collaborator (GitHub CLI) gh repo add-collaborator [repository] [username]

Collaboration Workflow

# Get latest changes git fetch origin git pull origin main # Create feature branch git checkout -b feature-name # Push branch to remote git push -u origin feature-name

Best Practices:

  • ✅ Always pull before starting new work
  • ✅ Use feature branches for new changes
  • ✅ Write clear commit messages
  • ⚠️ Avoid working directly on main branch
  • ⚠️ Resolve conflicts before merging

Pull Requests

# Create PR (GitHub CLI) gh pr create --title "Feature title" --body "Description" # Review PR gh pr checkout [PR-number] gh pr review [PR-number] --approve
